## Issue Summary
Add right-to-left (RTL) language support for the mobile app. Needed by GP MENA and requested by ANBT. A comment notes "R-to-L after beta."

## Analysis
**Significant work already done on the gh271 branch.** Current state:

### Already implemented:
- `android:supportsRtl="true"` in AndroidManifest.xml
- `isRTLLocale` constant in `src/constants.ts` for identifying RTL locales
- Arabic locale file (`locales/ar.json`) with computer-translated content for testing
- RTL-aware arrow icons in `src/components/review-culture-event.tsx` (prev/next question buttons)
- `direction: 'ltr'` fixes on timeline/progress bars to prevent mirroring:
  - `src/components/dual-recorder.tsx` (progressBar style)
  - `src/components/playlist.tsx` (progressContainer style)
  - `src/components/review-culture-event.tsx` (playbackBarContainer style)
- `localize.text()` wrapping throughout the app for all user-facing strings
- Localization context and locale switching infrastructure

### Still needed:
1. **I18nManager integration in app.tsx**: Force RTL layout when an RTL locale is selected (`I18nManager.forceRTL(true)` + app restart via `RNRestart`)
2. **Settings screen RTL toggle**: Allow switching to RTL locale from settings with app restart
3. **Comprehensive RTL icon review**: Audit all directional icons (arrows, chevrons) across all screens for RTL-awareness
4. **Text alignment audit**: Ensure text inputs, placeholders, and labels align correctly in RTL
5. **Flexbox layout audit**: Check all `flexDirection: 'row'` layouts for correct RTL behavior (some should mirror, timelines should not)
6. **Testing with actual RTL content**: Test with Arabic locale on device

## Implementation Plan
1. Add `I18nManager.forceRTL()` and `I18nManager.allowRTL()` calls in app initialization (`src/app.tsx`) based on the selected locale
2. Add `RNRestart` to restart the app when RTL/LTR mode changes
3. Add Arabic (or other RTL language) to the locale picker in Settings
4. Audit and fix remaining directional icons across all screens
5. Audit `flexDirection: 'row'` layouts that should NOT mirror (timelines already fixed; check for others like number inputs, time displays)
6. Test text input fields (`writingDirection` style property) for RTL alignment
7. Test navigation header back buttons for RTL direction
