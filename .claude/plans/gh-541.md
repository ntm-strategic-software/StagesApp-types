## Issue Summary
Add a "What to Listen For" overlay to the Playlist screen. A button (ear icon) toggles a scrollable markdown overlay between the top row and bottom controls. The content comes from `playlistListenFor(unit)` in stagesapp-types, keyed to the user's current CLA unit. The overlay must not affect playback, and scroll position should persist within a session but reset when re-entering the screen.

## Analysis
Much of this feature is **already implemented**:
- **Playlist screen** (`src/components/playlist.tsx`): Ear icon button exists (lines 846-860), state management in place (lines 481-482), ScrollView overlay renders content (lines 888-890), DraggableFlatList hides when overlay shown.
- **stagesapp-types**: `playlistListenFor(unit)` function exists in `src/utils.ts` (lines 9-14). Data in `src/playlist-listen-for-data.ts` with Unit 1 content covering 10 listening categories.
- **User's CLA unit**: Available via `activeUserClaUnit` from Redux (line 469).
- **No markdown renderer**: Content renders as plain `<Text>` — markdown symbols (##, -, etc.) show as-is. No markdown library in package.json.

### What's still needed:
1. **Markdown rendering** — content displays raw markdown syntax instead of formatted text
2. **More unit content** — only Unit 1 exists; need entries for additional units
3. **Scroll position persistence** — per the issue requirements, scroll position should persist within a session but reset on re-entering the screen

## Implementation Plan
1. Add `react-native-markdown-display` (or similar) to package.json
2. Replace the plain `<Text>` in the ScrollView overlay (line 889) with a markdown rendering component
3. Add a `ScrollView` ref to track scroll position; persist in a ref that resets on component mount
4. Add content for additional units in `StagesApp-types/src/playlist-listen-for-data.ts` (coordinate with content authors)
5. Verify overlay doesn't affect playback controls
